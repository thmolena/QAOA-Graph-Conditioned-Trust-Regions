Query‑Efficient Quantum Approximate Optimization via Graph‑Conditioned Trust Regions

Molena Huynh

Preprint · 2026

Every objective evaluation in a variational quantum algorithm is purchased with repeated circuit executions. Graph‑Conditioned Trust Regions turn a learned Gaussian over QAOA angles into a complete search policy: a warm start, a step geometry, a feasible region, and a validation‑fit per‑instance budget, all fixed before the first query. When the effective budget can evaluate both mandatory seeds, the policy includes the strongest online‑cheap prior as a counted seed and, under exact evaluation, best‑so‑far selection protects its value. That prior is derived from supervised training targets; it is not free of offline label cost.

QAOA Optimization as a Query‑Limited Problem

Depth‑\(p\) QAOA prepares \(|\gamma,\beta\rangle\) from a cost Hamiltonian \(C\) and mixer \(B\), and a classical loop maximizes \[ F_G(\theta)=\langle\theta|C|\theta\rangle, \qquad \theta\in\mathbb{R}^{2p}. \] On hardware, this objective is estimated from Born‑rule samples rather than read out directly, so every optimizer query costs repeated state preparation, execution, and measurement.

Every method is scored against the same offline 98% reference target. Reaching the target costs the number of objective evaluations used; a miss costs the common 400‑call benchmark cap. Raw work and an explicit attainment flag are retained alongside that capped score, so an early stop cannot make a failed run appear cheap. On a fixed trajectory, truncation can only tie or worsen this shared‑cap score. The reference is privileged evaluation information, not a deployable stopping oracle, and its construction is excluded from online counts.

A Graph‑Conditioned Gaussian as a Search Policy

A graph isomorphism network with Laplacian spectral encodings maps each graph to a diagonal Gaussian over QAOA angles, \[ p(\theta\mid G)=\mathcal{N}\!\bigl(\mu(G),\Sigma(G)\bigr), \qquad \Sigma(G)=\operatorname{diag}\!\bigl(\sigma^2(G)\bigr). \] The mean initializes search. The standard deviations precondition its coordinate steps and define the Mahalanobis trust region \[ \mathcal{T}(G)=\{\theta:\|\Sigma^{-1/2}(\theta-c)\|_2\leq2\}. \] Every proposal is retracted onto this region in whitened coordinates.

The seed set contains the predicted mean, the parameter‑concentration angles, and score‑allocated Gaussian draws. Each evaluated seed costs one query, the region re‑centers on the best seed, and best‑so‑far selection preserves the best evaluated exact value. Comparison with both mandatory seeds requires enough effective budget to evaluate them. This distinguishes a learned search policy from a learned point initializer.

A Held‑Out Difficulty Score Prices Each Instance

The covariance sizes the search geometry. A separate linear calibrator is fit on held‑out validation residuals, never on test outcomes, and produces the difficulty score used for allocation. On the 48 test graphs, that score has Spearman \(\rho=0.671\) with realized warm‑start error (\(p=1.76\times10^{-7}\)). Separately, the Gaussian angle head has marginal per‑coordinate coverage ECE 0.101 over 192 residuals and observed coverage 0.901 at nominal 0.95; ECE does not calibrate the difficulty score. Removing the ranking penalty improves ECE to 0.026 and score correlation to 0.854 in the released ablation, so that loss term is not supported by this run.

Validation statistics standardize the score and map it monotonically to a seed count and evaluation cap before the first query. The mandatory mean and concentration seeds are protected only when the effective budget can evaluate them. The released two‑seed/full‑cap control changes both Gaussian seeding and the cap, so it cannot identify either component's causal contribution.

Benchmarking Graph‑Conditioned Trust Regions

The released run studies unweighted MaxCut at \(n=14\), depth \(p=2\), with 240/80/48 train/validation/test graphs from four random‑graph families. Schema‑2 source data retain capped cost, raw evaluations, attainment, primary and leave‑one‑family‑out instance rows, an implementation fingerprint, and artifact hashes. The exact‑statevector evidence stops at \(n=16\); no device result or larger‑scale claim is made. Because this revision inspected the test results while correcting the protocol and claims, confirmation requires a new prospectively untouched audit set.

In‑Distribution Cost and Attainment

GCTR reaches \(48/48\) targets at capped cost \(15.0\pm7.8\) calls and mean exact expectation ratio 0.858. The supervised concentration heuristic is online‑cheaper, also reaches \(48/48\), and records \(12.33\pm6.45\) (paired \(p=2.6488\times10^{-4}\)). TQA reaches \(48/48\) at \(18.48\pm13.48\), with no established cost difference from GCTR (\(p=0.0683\)). Random restarts, k-NN, and the GNN point initializer reach \(37/48\), \(42/48\), and \(20/48\), with capped costs \(203.9\pm152.9\), \(52.94\pm131.93\), and \(264.42\pm164.02\), respectively.

Angle Coverage and Difficulty Ranking

The Gaussian heads attain coverage ECE 0.101 over 192 standardized residuals. The validation‑fit difficulty score ranks realized warm‑start error with \(\rho=0.671\) (\(p=1.76\times10^{-7}\), \(n=48\)). Coverage calibration and difficulty ranking are distinct diagnostics; neither is a calibrated probability of target attainment.

Leave‑One‑Family‑Out Shift

Pooled across four held-out graph families, GCTR reaches \(39/48\) targets at capped cost 89.3. The heuristic reaches \(40/48\) at 86.5 (\(p=0.832\) versus GCTR), while the point initializer reaches \(38/48\) at 125.0 (\(p=0.00509\)). The two‑seed/full‑cap control reaches \(41/48\) at 73.75 (\(p=0.0355\)). These exploratory paired tests share four fitted family clusters, and the composite control does not isolate learned allocation from its coupled policy changes.

Cross‑Size Transfer Through 16 Qubits

A model trained at \(n=14\) has an 8.47--13.95\(\times\) lower mean capped cost than random restarts for \(n=8,10,12,14,16\). GCTR costs are 11.6, 20.1, 13.4, 15.4, and 15.2; heuristic costs are 10.9, 21.8, 12.1, 12.5, and 10.5, with identical attainment at every size. Thus the heuristic is cheaper at four of five sizes; GCTR is slightly cheaper only at \(n=10\). The random/GCTR ratio is arithmetic benchmark cost, not wall‑clock or hardware speedup, and the exact‑statevector boundary prevents extrapolation beyond \(n=16\).

Reproducible Research Package

The repository root is the single installable source for the specops-gctr distribution. The release contains the exact‑statevector objective, graph generator, trainable predictor, held‑out error head, search policies, statistics, source data, figure and table generators, and a release test suite. Schema‑2 validation checks the implementation fingerprint and artifact hashes. The package does not ship a pretrained checkpoint. Fitting currently requires exact‑simulation calibration instances and graph‑only prediction still uses dense linear algebra. A backend‑neutral callback API applies the same counted policy to QPU runtimes, emulators, or external scalar objectives without treating that integration path as hardware evidence. 

python -m pip install "specops-gctr @ git+https://github.com/thmolena/QAOA-Graph-Conditioned-Trust-Regions.git"
gctr-optimize --family er --n 10 --budget 100 --json

# Replot and tests require a repository checkout:
gctr-reproduce --replot-only --source-data-dir manuscript/source_data
gctr-reproduce --validate-only --manuscript-dir manuscript
python -m pytest tests -q
